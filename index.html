Server Information

DNS Zones

Execution Mode

Execute on server (live mode) When enabled, policies are created directly on the DNS server instead of generating PowerShell commands.

Client Subnets

▼

Named subnets used in policy criteria (e.g., -ClientSubnet "EQ,NorthAmericaSubnet").

Zone Scopes

▼

Zone scopes allow ALLOW policies to route queries to different record sets within the same zone.

Recursion Scopes

▼

Control recursion behavior per client group. The default scope "." can be modified but not deleted.
