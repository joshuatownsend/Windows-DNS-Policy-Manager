## Findings considered and rejected (so they aren't re-audited)

- **postcss override "not applied"** — REJECTED. The lockfile resolves
  `node_modules/postcss` to `8.5.12` (≥ the `>=8.5.10` override floor), with no
  nested styled-jsx copy. The override added in commit `cdddbec` works; the audit
  subagent misread a semver range as a resolved version.
- **DPAPI credential files need explicit ACLs** — REJECTED as low value.
  `%LOCALAPPDATA%\DNSPolicyManager\credentials` is already per-user, and DPAPI is
  CurrentUser-scoped, so the ciphertext is useless to other users. Defense-in-depth
  at best; not worth an executor's time.
- **"MCP exposes writes" / "bridge cmdlet calls are injectable"** — VERIFIED SAFE
  (no action). All 31 MCP tools are read-only, `/api/execute` is not exposed via
  MCP, and typed bridge handlers use splatted parameters. The path-traversal guards
  (`Assert-SafeId` = `^[a-zA-Z0-9._-]+$`, `Assert-SafeFileName` =
  `Path.GetFileName` equality) are robust — the README's claim holds.
- **Wizards non-null assertions** (`wizards/page.tsx`) — not planned. React's guard
  at the top of the render path makes the null case unreachable in practice; very
  low real risk.
