## Findings NOT yet planned (surfaced in the audit or during execution)

- **F9 — Runspace-pool shared hashtables mutated without synchronization**
  (`server/bridge.ps1:4825-4843`; `SessionCredentials`/`BpaJobs`/`ResolverJobs`).
  Real race risk under concurrency, but mitigated by single-user localhost usage.
  MED confidence/MED effort. Revisit if the bridge ever serves concurrent clients.
  (Note: plan 006 added `$script:RequestCimSessions` following this same sharing
  pattern — if F9 is ever addressed with a thread-safe object, revisit 006's list.)
- **F10 — Raw `$_.Exception.Message` returned to clients** (~60 sites). Info
  disclosure; low severity for a localhost tool. S effort if batched.
- **F11 — God objects**: `server/bridge.ps1` (4,874 lines) and
  `dns-manager/src/app/zones/page.tsx` (1,450). L effort, MED risk, weak
  verification story. **Do not decompose the bridge until a Pester test harness
  exists** (a prerequisite plan).
- **F12 — Raw `serverHostname` in the primary/secondary wizard** (surfaced during
  plan 004 execution). `-MasterServers "${serverHostname}"` in the `primarysecondary`
  scenario (`dns-manager/src/wizards/command-generator.ts` ~line 276 and the MCP
  twin `mcp-server/src/tools/command-gen.ts` ~line 295) is interpolated **without**
  `psEscape()`, because it sits inside the `switch` body that plan 004 was scoped
  not to touch. Same copy-paste foot-gun class as F4 (copy-pasteable command only;
  the execution path is splatted/safe). S effort, LOW severity. Fix when the switch
  bodies are next edited, or fold into the shared-module extraction (plan 004
  Maintenance notes).
- **F13 — MCP server didn't compile on `main`** (surfaced during the plan 005
  re-run) — **FIXED in PR #46**. Dependabot's TS 5.9→6.0 + Zod bumps broke
  `mcp-server` build: 7× `process` TS2591 (needed `"types": ["node"]` in
  `mcp-server/tsconfig.json`) and a `z.record` arity error at `command-gen.ts:372`
  (newer Zod needs a key schema). It went unnoticed precisely because the MCP server
  had **no CI** — which is what plan 005 adds. Both the fix and the guardrail ship
  together in #46.

Execution follow-up (not a bug): `Test-CommandAllowed` (plan 001) intentionally
omits `Where-Object`/`ForEach-Object` from its allowlist. No current UI path sends
those through `/api/execute` (the reviewer confirmed `api.execute` is only invoked
from `app/create/page.tsx` for `Add-DnsServer*` commands). If a free-form execution
UI is ever added, those two can be added to `$allowedExact` safely — commands nested
inside their script blocks are still walked and validated by the AST check.

## Findings considered and rejected (so they aren't re-audited)

- **postcss override "not applied"** — REJECTED. The lockfile resolves
  `node_modules/postcss` to `8.5.12` (≥ the `>=8.5.10` override floor), with no
  nested styled-jsx copy. The override added in commit `cdddbec` works; the audit
  subagent misread a semver range as a resolved version.
- **DPAPI credential files need explicit ACLs** — REJECTED as low value.
  `%LOCALAPPDATA%\DNSPolicyManager\credentials` is already per-user, and DPAPI is
  CurrentUser-scoped, so the ciphertext is useless to other users. Defense-in-depth
  at best; not worth an executor's time.
- **"MCP exposes writes" / "bridge cmdlet calls are injectable"** — VERIFIED SAFE
  (no action). All 31 MCP tools are read-only, `/api/execute` is not exposed via
  MCP, and typed bridge handlers use splatted parameters. The path-traversal guards
  (`Assert-SafeId` = `^[a-zA-Z0-9._-]+$`, `Assert-SafeFileName` =
  `Path.GetFileName` equality) are robust — the README's claim holds.
- **Wizards non-null assertions** (`wizards/page.tsx`) — not planned. React's guard
  at the top of the render path makes the null case unreachable in practice; very
  low real risk.
